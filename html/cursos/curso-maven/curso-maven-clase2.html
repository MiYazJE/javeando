Curso de Maven

Propiedades del proyecto maven

Todo proyecto Maven necesita la configuración de el fichero pom.xml, que es donde se definen todas las propiedades (dependencias, pluggins, encoding…)
Este podria ser el fichero pom de cualquier proyecto: 

                        
<project xmlns="http://maven.apache.org/POM/4.0.0" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
  xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
  <modelVersion>4.0.0</modelVersion>
 
  <groupId>com.javeando.app</groupId>
  <artifactId>my-app</artifactId>
  <version>1.0-SNAPSHOT</version>
 
  <properties>
    <maven.compiler.source>1.7</maven.compiler.source>
    <maven.compiler.target>1.7</maven.compiler.target>
  </properties>
 
  <dependencies>
    <dependency>
      <groupId>junit</groupId>
      <artifactId>junit</artifactId>
      <version>4.12</version>
      <scope>test</scope>
    </dependency>
  </dependencies>
</project>

Qué acabamos de hacer?

Hemos definido en las propiedades del proyecto que compile bajo la versión 1.7 de java y también hemos añadido la herramienta de Jboss para realizar pruebas unitarias.

Construir el proyecto

Las opciones que nos da Maven despues de haber creado un arquetipo son:
  • mvn compile - compila el proyecto y deja el resultado en target/classes
  • mvn test – compila los test y los ejecuta
  • mvn package – empaqueta el proyecto y lo dejará en taget/autentiaNegocio-1.0-SNAPSHOT.jar
  • mvn install – guarda el proyecto en el repositorio
  • mvn clean – borra el directorio de salida (target)
Estas tareas son siempre las mismas así que cuando un desarrollador salte de un proyecto a otro siempre sabrá hacer estas cosas.

Maven define un ciclo de vida por lo que al ejecutar un objetivo, antes se ejecutan los sus antecesores en el ciclo de vida. Por ejemplo, si ejecutamos package, antes se ejecutará compile y test.

Para saber mas sobre esto dejo un link.

Vamos a realizar la etapa de package que es la que en el ciclo de la vida englobaria a todas las anteriores. 

mvn package
